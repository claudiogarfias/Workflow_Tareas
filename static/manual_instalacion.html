Manual de Instalación y Publicación en IIS

Guía técnica procedimental detallada para el despliegue del aplicativo "Workflow de Tareas v2" bajo servidor corporativo.

Servidor Destino:
LH-PENTANA-01
Dominio (DNS):
losheroes.losheroes.cl
Sistema Operativo:
Windows Server 2019
Servidor Web:
IIS (Internet Information Services)
Motor de Base de Datos:
SQLite (workflow_tareas.db)
Tecnología Backend:
Python 3.10+ / Flask / Waitress

Requisitos Previos

Antes de iniciar con el procedimiento de instalación en el servidor LH-PENTANA-01, verifique que cuenta con las siguientes condiciones:

  • Acceso de Administrador: Credenciales de administrador local en el servidor Windows Server 2019.
  • Servidor de IIS Activado: El rol de "Servidor Web (IIS)" debe estar habilitado con el servicio de herramientas de administración del IIS.
  • Acceso a Red / Internet: Habilitación de navegación para descarga del instalador de Python y el módulo HttpPlatformHandler, o disponer de los archivos instaladores locales.
  • Puerto Disponible: Puerto HTTP 80 (o HTTPS 443) libre y configurado en el Firewall para el dominio losheroes.losheroes.cl.
Nota Arquitectónica
La aplicación se compone de un backend Flask y una base de datos SQLite embebida. Para producción en IIS, se utilizará Waitress como servidor WSGI (puente local) y el módulo HttpPlatformHandler de Microsoft para administrar los subprocesos de Python de manera transparente.

1Instalación de Python en el Servidor

En el servidor LH-PENTANA-01, ejecute los siguientes pasos:

  1. Descargue el instalador oficial de Python (versión 3.10 o superior) desde python.org (se recomienda la versión 3.10.x de 64 bits).
  2. Ejecute el instalador descargado como Administrador.
  3. CRÍTICO: En la primera ventana del instalador, marque la casilla "Add Python 3.xx to PATH". Esto permitirá la ejecución global de Python.
  4. Seleccione "Install Now" o elija una ruta personalizada si su organización lo requiere (ej. C:\Program Files\Python310).
  5. Abra una consola de PowerShell (o CMD) como Administrador y verifique la correcta instalación ejecutando:
python --version
pip --version

2Instalación de Microsoft HttpPlatformHandler

El módulo HttpPlatformHandler permite a IIS gestionar procesos de backend de forma ágil y redirigir el tráfico del puerto de IIS (80/443) hacia la aplicación Python de forma automática.

  1. Descargue el instalador oficial de la extensión de IIS desde la página de Microsoft:
    • HttpPlatformHandler en IIS.net o el enlace directo al MSI de 64 bits.
  2. Ejecute el instalador httpPlatformHandler_amd64.msi en el servidor LH-PENTANA-01.
  3. Siga el asistente de instalación estándar (Siguiente, Siguiente, Finalizar).
  4. Reinicie el servicio de IIS para aplicar la extensión ejecutando en la consola de comandos de administración:
iisreset

3Preparación del Directorio del Aplicativo

Es indispensable estructurar los archivos de la aplicación en el directorio raíz de publicaciones de IIS.

  1. Cree una carpeta dedicada para la aplicación en la ruta clásica de IIS en Windows Server:
    C:\inetpub\wwwroot\WorkflowTareas
  2. Copie y pegue en este directorio los siguientes archivos y carpetas del proyecto:
    • server.py (Backend Flask principal)
    • run_waitress.py (Arrancador de producción WSGI)
    • web.config (Archivo de configuración de IIS)
    • requirements.txt (Archivo de dependencias de Python)
    • workflow_tareas.db (Base de datos SQLite inicial)
    • Carpeta static/ (Archivos frontend: HTML, JS, CSS y componentes gráficos)
Importante
Evite copiar carpetas como .venv, .git o archivos de desarrollo local que aumenten el peso innecesariamente. Las dependencias en el servidor deben instalarse de forma limpia y directa.

4Instalación de Dependencias de Python

Una vez copiados los archivos, es necesario instalar las bibliotecas de Python en el servidor para que el backend funcione.

  1. Abra la consola de Símbolo del Sistema (CMD) o PowerShell como Administrador.
  2. Navegue al directorio de la aplicación:
cd C:\inetpub\wwwroot\WorkflowTareas
  1. Ejecute la instalación de dependencias corporativas, incluyendo waitress (servidor WSGI para producción Windows):
pip install -r requirements.txt waitress

5Configuración de Permisos en la Carpeta (Paso Crítico)

Al utilizar una base de datos SQLite (workflow_tareas.db), la base de datos se guarda en un archivo de disco ordinario. Para que la aplicación pueda registrar cambios, tareas, auditorías y logs, el usuario que ejecuta el pool de aplicaciones en IIS (identidad del sitio web) requiere permisos de Modificación y Escritura.

Riesgo Operativo Crítico
Si omite este paso, el aplicativo cargará la pantalla inicial, pero al intentar crear, editar o eliminar tareas, el servidor arrojará un error 500 Internal Server Error debido a que SQLite no podrá escribir en el archivo ni generar los archivos de transacciones temporales (.db-journal o .db-wal) en el directorio.

Siga estos pasos al pie de la letra para asignar los permisos necesarios:

  1. Navegue mediante el Explorador de Archivos de Windows a C:\inetpub\wwwroot.
  2. Haga clic secundario sobre la carpeta WorkflowTareas y seleccione Propiedades.
  3. Vaya a la pestaña de Seguridad y haga clic en el botón Editar...
  4. En la ventana de permisos, pulse el botón Agregar...
  5. Escriba el nombre del grupo de usuarios locales de IIS: IIS_IUSRS y pulse en Comprobar nombres. El sistema debe subrayar el nombre confirmando su existencia.
  6. Haga clic en Aceptar.
  7. Seleccione IIS_IUSRS de la lista y en la ventana de permisos inferior, marque la casilla de permitir para "Modificar" y "Escritura". (La lectura y ejecución se marcarán automáticamente).
  8. Haga clic en Aplicar y luego en Aceptar en todas las ventanas.

6Creación y Publicación del Sitio Web en IIS

Siga esta secuencia para dar de alta la aplicación en IIS utilizando el dominio corporativo requerido:

  1. Abra el Administrador de Internet Information Services (IIS) desde las herramientas administrativas de Windows o ejecutando inetmgr.
  2. En el panel izquierdo, despliegue el nombre del servidor (LH-PENTANA-01) y haga clic derecho sobre Sitios (Sites). Elija la opción Agregar sitio web... (Add Website...).
  3. Complete los datos en el cuadro de diálogo:
    • Nombre del sitio: WorkflowTareas
    • Ruta de acceso física: Seleccione la carpeta creada C:\inetpub\wwwroot\WorkflowTareas
    • Enlace:
      • Tipo: http
      • Dirección IP: Todas las no asignadas (o la IP específica del servidor LH-PENTANA-01)
      • Puerto: 80
      • Nombre de host: losheroes.losheroes.cl
  4. Asegúrese de que la casilla "Iniciar sitio web inmediatamente" esté marcada.
  5. Haga clic en Aceptar.

Resolución DNS / Mapeo en Clientes

Configuración de Red
Para que los usuarios puedan acceder mediante el dominio corporativo http://losheroes.losheroes.cl, se debe registrar una entrada tipo A en el DNS corporativo de la organización que apunte el dominio losheroes.losheroes.cl a la dirección IP física del servidor LH-PENTANA-01.

Para pruebas locales rápidas en el servidor o cliente: Puede editar el archivo hosts de Windows (C:\Windows\System32\drivers\etc\hosts) y añadir la línea:
IP_DEL_SERVIDOR losheroes.losheroes.cl

Resolución de Problemas (Diagnóstico)

Si la aplicación muestra errores al cargarse (como 500 Internal Server Error o 502 Bad Gateway), verifique los siguientes puntos de control de bitácora:

1. Archivo python.log

El módulo HttpPlatformHandler redirigirá la consola estándar de Python directamente a un archivo log para diagnosticar errores. Revise el archivo generado automáticamente en:

C:\inetpub\wwwroot\WorkflowTareas\python.log

Los errores comunes reportados en este archivo suelen ser:

  • ModuleNotFoundError: No module named 'flask': Ocurre si las dependencias no se instalaron de forma global o si el script `web.config` no localiza el ejecutable correcto de Python. Modifique el parámetro processPath en web.config con la ruta completa a su binario de Python.
  • sqlite3.OperationalError: unable to open database file: Ocurre si omitió el paso de otorgar permisos de modificación a IIS_IUSRS.

2. Ajuste manual de web.config

Si Python se instaló en una ruta no estándar en LH-PENTANA-01, abra el archivo web.config con permisos de administrador y especifique la ruta exacta en processPath:

<!-- Ejemplo con ruta absoluta en web.config -->
<httpPlatform stdoutLogEnabled="true" stdoutLogFile=".\python.log" startupTimeLimit="60" 
              processPath="C:\Program Files\Python310\python.exe" 
              arguments="run_waitress.py">
